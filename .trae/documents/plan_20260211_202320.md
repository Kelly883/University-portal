# Vercel & Neon Configuration Plan

## 1. Dependency Management
- **Objective**: Add necessary packages for production deployment and database connection.
- **Actions**:
    - Add `dj-database-url` to `requirements.txt` (for parsing Neon's `DATABASE_URL`).
    - Add `whitenoise` to `requirements.txt` (for serving static files on Vercel).
    - Add `gunicorn` (optional, but standard for production).

## 2. Django Settings Configuration (`settings.py`)
- **Objective**: Configure Django to work with Neon and run on Vercel.
- **Actions**:
    - **Database**: Replace the manual `DATABASES` dict with `dj_database_url.config()` to automatically parse the `DATABASE_URL` environment variable provided by Neon/Vercel.
    - **Static Files**: Configure `Whitenoise` middleware and storage to serve static assets efficiently.
    - **Allowed Hosts**: Update `ALLOWED_HOSTS` to accept Vercel's domain (`.vercel.app`) and `127.0.0.1`.
    - **CSRF**: Update `CSRF_TRUSTED_ORIGINS` to allow the Vercel URL.

## 3. Vercel Deployment Configuration
- **Objective**: Tell Vercel how to build and run the Django app.
- **Actions**:
    - Create `vercel.json` configuration file defining the build and route handling (pointing to `wsgi.py`).
    - Ensure `wsgi.py` exposes the `app` variable if required by the specific Vercel runtime configuration (though standard Django `application` usually works, explicit mapping is safer).

## 4. Git Sync
- **Objective**: Push these changes to GitHub so Vercel can trigger a deployment.
- **Actions**:
    - `git add .`
    - `git commit -m "Configure for Vercel deployment with Neon DB"`
    - `git push origin main`
