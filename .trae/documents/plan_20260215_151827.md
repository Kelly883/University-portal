# Deployment Fix Plan: `npm ci` Failure

## 1. Analysis
*   **Error**: `npm error code EUSAGE` / `npm ci` can only install packages when your `package.json` and `package-lock.json` are in sync.
*   **Cause**: We recently updated `package.json` (likely adding `ts-node` for the seed script) but did not run `npm install` locally to update `package-lock.json`. Thus, the lockfile is out of sync, causing `npm ci` (clean install) to fail on the build server.
*   **Missing Packages**: `ts-node`, `acorn-walk`, `arg`, `create-require`, etc. (all dependencies of `ts-node`).

## 2. Solution
We need to update `package-lock.json` to reflect the changes in `package.json`.

### Steps:
1.  **Run `npm install`**: This will generate the correct `package-lock.json` containing the missing dependencies.
2.  **Commit & Push**: Push the updated lockfile to GitHub.

## 3. Execution
1.  Run `npm install` in the terminal.
2.  Run `git add package-lock.json`.
3.  Commit and push.
