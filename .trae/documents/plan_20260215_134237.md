# Deployment Fix Plan: Next.js Font Import Error

## 1. Analysis
- **Error**: `Type error: Module '"next/font/google"' has no exported member 'Geist'.`
- **Cause**: The `Geist` font was introduced in newer versions of Next.js (likely 15+ or specific 14.x releases with experimental flags). We are running `Next.js 14.2.35` which might not have `Geist` available by default or under that specific export, OR the template code assumed a newer version.
- **Context**: The `layout.tsx` file is trying to import `Geist` and `Geist_Mono` from `next/font/google`, but they aren't found.

## 2. Solution
Replace the `Geist` font imports with the standard `Inter` font, which is universally supported in Next.js 14. This is a safe fallback to ensure the build passes.

### Steps:
1.  **Modify `src/app/layout.tsx`**:
    - Remove imports for `Geist`, `Geist_Mono`.
    - Import `Inter` from `next/font/google`.
    - Replace usage of `geistSans` and `geistMono` with `inter`.
    - Update the `body` class name to use the `inter` class.

## 3. Implementation Details
```typescript
import type { Metadata } from "next";
import { Inter } from "next/font/google"; // Switch to Inter
import "./globals.css";

const inter = Inter({ subsets: ["latin"] }); // Initialize Inter

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body className={inter.className}>{children}</body> {/* Use Inter */}
    </html>
  );
}
```
