# Migration from Django Templates to Next.js

## 1. Backend Transformation (Django -> DRF API)
- **Objective**: Convert existing Django Views (which render templates) into REST API endpoints using Django Rest Framework (DRF).
- **Steps**:
    1.  **Install DRF**: Add `djangorestframework`, `django-cors-headers` to `requirements.txt`.
    2.  **Configure Settings**: Update `INSTALLED_APPS` and `MIDDLEWARE` (CORS).
    3.  **Create Serializers**:
        -   `UserSerializer`, `CourseSerializer`, `EnrollmentSerializer`, etc.
    4.  **Create API Views**:
        -   Replace `ListView`/`DetailView` with `generics.ListCreateAPIView`, etc.
        -   Create `api/` endpoints in `urls.py`.
    5.  **Authentication**: Configure Token Authentication (SimpleJWT or Session for now, assuming JWT for Next.js).

## 2. Frontend Creation (Next.js)
- **Objective**: Create a new Next.js project to replace the Django Template frontend.
- **Steps**:
    1.  **Initialize Next.js**: `npx create-next-app@latest frontend`.
    2.  **Setup UI**: Install Tailwind CSS or Bootstrap (matching current design).
    3.  **Implement Pages**:
        -   `/login`, `/register`: Auth forms.
        -   `/dashboard`: Role-based dashboard (fetching from Django API).
        -   `/courses`: List and detail views.
    4.  **State Management**: Use Context API or libraries for Auth state.

## 3. Integration & Deployment
- **Objective**: Connect Frontend to Backend and deploy.
- **Steps**:
    1.  **CORS**: Ensure Django allows requests from the Next.js domain.
    2.  **Env Vars**: Set API URL in Next.js.
    3.  **Vercel Config**: Update `vercel.json` to handle both or deploy separately (typically separate deployments: one for API, one for Next.js, or Monorepo).
