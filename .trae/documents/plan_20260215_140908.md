# Production Readiness Plan

## 1. Environment & Configuration Check
*   **Goal**: Ensure all necessary environment variables are documented and the app is configured for production.
*   **Action**:
    *   Review `.env.example` to ensure all keys (DB, Auth, APIs) are present.
    *   Verify `next.config.mjs` for production settings (e.g., image domains if needed).
    *   Check `railway.json` for correct build/start commands.

## 2. Error Handling & UI Polish
*   **Goal**: Prevent "ugly" crashes and ensure a smooth user experience.
*   **Action**:
    *   Create a global `error.tsx` in `src/app` to catch unhandled runtime errors.
    *   Create a global `not-found.tsx` for 404s.
    *   Ensure `loading.tsx` exists for smooth transitions.

## 3. Database Optimization
*   **Goal**: Ensure Prisma is ready for production loads.
*   **Action**:
    *   Verify `schema.prisma` indexes (already present on foreign keys implicitly, but good to double-check).
    *   Ensure connection pooling is enabled in the `DATABASE_URL` (Neon uses pooling by default, but we should verify the `sslmode=require` parameter).

## 4. Security Audit (Basic)
*   **Goal**: Basic security hardening.
*   **Action**:
    *   Verify `middleware.ts` matcher covers all sensitive routes.
    *   Ensure `next-auth` secret is required in prod.

## 5. Build Verification
*   **Goal**: Ensure the app builds locally without errors before pushing.
*   **Action**: Run `npm run build` locally.

## Execution Steps
1.  **Add Error/404 Pages**: Create `src/app/not-found.tsx` and `src/app/error.tsx`.
2.  **Add Loading State**: Create `src/app/loading.tsx`.
3.  **Verify Middleware**: Double-check `matcher` in `middleware.ts`.
4.  **Local Build**: Run a build check.
5.  **Commit**: Push final polish.
