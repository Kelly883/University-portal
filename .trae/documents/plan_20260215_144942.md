# Login Page Redesign Plan

## 1. Analysis & Requirements
*   **Goal**: Replace the default/existing login page with the user's provided "Titan University" design.
*   **Tech Stack**: Next.js App Router, Tailwind CSS, NextAuth (Credentials).
*   **Design Assets**:
    *   **Colors**: `#2D3436` (Primary), `#1B263B` (Accent), `#C5A059` (University Gold).
    *   **Fonts**: Inter, Barlow Condensed, Playfair Display.
    *   **Icons**: Material Symbols Outlined (via Google Fonts).
    *   **Layout**: Centered card with "University" branding, background patterns, and specific input styling.

## 2. Implementation Strategy
*   **Location**: `src/app/login/page.tsx`.
*   **Dependencies**:
    *   Need to load Google Fonts (Inter, Barlow Condensed, Playfair Display) via `next/font/google` or `layout.tsx` to optimize performance, rather than using the raw HTML link from the snippet.
    *   Need to integrate the `authenticate` logic (Server Action or Client Component handler) into the new form.
*   **Component Structure**:
    *   `LoginPage` (Server Component or Client Component wrapper).
    *   `LoginForm` (Client Component): Handles state (email, password, error), submission, and validation.

## 3. Step-by-Step Plan
1.  **Configure Fonts**: Add `Barlow Condensed` and `Playfair Display` to `src/app/layout.tsx` (or a dedicated font utility) alongside `Inter`.
2.  **Update Tailwind Config**: Add the custom colors (`university-gold`, `accent`, etc.) and font families to `tailwind.config.ts`.
3.  **Create Login Page**:
    *   Translate the provided HTML/CSS into React/JSX.
    *   Replace the static `<form>` with a functional NextAuth login form.
    *   Handle "Forgot Key" and "Request Access" links (stub or link to real routes).
4.  **Integration**:
    *   Ensure the form calls `signIn("credentials", ...)` from `next-auth/react`.
    *   Handle loading states and error messages (e.g., "Credential format unrecognized") dynamically.

## 4. Execution Details
*   **File**: `src/app/login/page.tsx`
*   **Logic**:
    *   `const [errorMessage, setErrorMessage] = useState("")`
    *   `const [isPending, startTransition] = useTransition()`
    *   `action={dispatch}`

## 5. Refinement
*   The provided HTML uses `cdn.tailwindcss.com`. I will convert these arbitrary values to standard Tailwind classes or extended config values to ensure maintainability.
*   The "academic-pattern" CSS will be added to `globals.css` or a CSS module.

## Execution Steps
1.  **Tailwind Config**: Update colors/fonts.
2.  **Globals CSS**: Add the pattern styles.
3.  **Login Component**: Build the page with the new design and wire up NextAuth.
